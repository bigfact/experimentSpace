# 接金币活动

## H5 页面方案

1. H5 页面分为 APP 版本和浏览器版本
2. 只有一个 H5 页面，在页面中判断是 APP 访问还是浏览器访问

## 流程

### 微信或其他浏览器
* 进入落地页
    * 页脚有提示下载 APP 的浮层
* 点击进入游戏
* 检查活动时间
    * 若活动结束，则提示，且不进入游戏
* 游戏倒计时
* 游戏中
* 游戏结束，检查是否绑定微信服务号
    * 若绑定了微信服务号，则跳过输入手机号流程（老用户）
    * 若未绑定，则进入输入手机号流程
* 游戏结束，未绑定微信服务号，弹框提示输入手机号码领取金币
    * 若手机号错误，则提示
* 输入验证码
    * 若验证码错误，则提示
* 提示成功收集到金币
    * 老用户，若没有参加过活动，提示分享
    * 老用户，若已经参加过活动并领取了金币，则提示，最开始参加的一次收集到的金币数
    * 老用户，若已经参加过活动并领取了金币，且达到 500 个，则提示
    * 新用户，提示下载 APP 查看奖励，无分享，并给分享的老用户加金币

### APP
* 进入落地页
* 检查登录状态
    * 若未登录，则提示
    * 若登录且玩过该游戏，则提示
* 点击进入游戏
* 游戏倒计时
* 游戏结束，提示成功收集到金币
    * 老用户，若没有参加过活动，提示分享
    * 老用户，若已经参加过活动并领取了金币，则提示，最开始参加的一次收集到的金币数
    * 老用户，若已经参加过活动并领取了金币，且达到 500 个，则提示

### 注意
* 不显示哪些新用户帮助老用户收集了金币

## 数据
* 用户
    * 手机号
    * token ？
    * 头像 ？
* 游戏
    * 自己的金币数
    * 其他用户帮自己收集的金币数
